Q1: How Does JavaScript Works?

Answer: JavaScript is mainly a Scripting language. Which is mainly worked on client side. It's a programming language worked on web browser by DOM or Document Object Model. The code passing through a in-build compiler and the compiler translate the code into bytecode which is machine readable.
When you place JavaScript code on a web page and load the page then automatically a virtual machine interpreted the code and marge with DOM. The virtual machine name is V8 machine which is a build in IDE implemented by Google Chrome. JavaScript is single-threated programming language. Which can exicute code asynchronously. It's mainly exicute code line by line from top to bottom approch. It's a human readable high level programming language.

Q2: Difference Between Local Storage vs Session Storage?

Answer: Local storage and session storage both are mainly used for tracking the useful information from a website. Supoose when a user want to buy something his/her previous record will be saved on that. But the main difference on local and session storage is - Local Storage: Local storage is mainly a read only property on Dev-Tool API. It's stored the data permanently and when the user back on the page after a long time its stored the data safely and re-viewd the storage on the page. So there has no fear to losing the data. It's mainly stored the data object wise and render the data with stringify. And shows it on Web Page.
Session Storage: Session storage working exactly like local storage but the difference is it's stored the data session wise that means when user close the page or the Tab then the session storage will be removed as with the page.
